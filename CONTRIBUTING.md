# Contributing to AMEP

Thank you for considering contributing to AMEP!

[中文版](#中文版)

## 🐛 Reporting Bugs

1. Search [Issues](../../issues) to check if the problem has already been reported
2. If not, create a new Issue including:
   - Clear title
   - Steps to reproduce
   - Expected behavior vs actual behavior
   - Environment info (Node.js version, OS, etc.)

## 🔧 Submitting Code

1. Fork this repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Commit your changes: `git commit -m 'Add some feature'`
4. Push the branch: `git push origin feature/your-feature`
5. Create a Pull Request

## 📋 Code Standards

- Use TypeScript
- Follow existing code style
- Add necessary comments
- Update relevant documentation

## 🛠️ Development Guide

```bash
# Install dependencies
npm install

# Build
npm run build

# Test
npm test
```

## 中文版

感谢你考虑为 AMEP 做贡献！

### 报告 Bug

1. 在 [Issues](../../issues) 中搜索是否已有相同问题
2. 如果没有，创建新 Issue，包含：
   - 清晰的标题
   - 复现步骤
   - 期望行为 vs 实际行为
   - 环境信息（Node.js 版本、操作系统等）

### 提交代码

1. Fork 本仓库
2. 创建功能分支：`git checkout -b feature/your-feature`
3. 提交更改：`git commit -m 'Add some feature'`
4. 推送分支：`git push origin feature/your-feature`
5. 创建 Pull Request

### 代码规范

- 使用 TypeScript
- 遵循现有的代码风格
- 添加必要的注释
- 更新相关文档

### 开发指南

```bash
npm install    # 安装依赖
npm run build  # 编译
npm test       # 测试
```
